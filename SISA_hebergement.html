Hébergement — comparatif & recommandations

Mise à jour : 2025-09-15 · focus sur site statique + intranet SISA

Recommandation rapide

SISA **Statique** (pages HTML, assets) : GitHub Pages pour staging → Cloudflare Pages en prod (DNS Cloudflare, Analytics, Turnstile si formulaire).
Option **Dynamique** (WordPress/PHP, email) : WHC au besoin pour clients qui exigent PHP/IMAP canadiens.

Comparatif synthétique

PlateformePoints fortsLimitesUsage recommandé
GitHub Pages Gratuit, sans pubs, déploiement par commit, parfait pour statique. Pas de backend; domaine `github.io` pour staging. Préprod/staging du site SISA; preview client.
Cloudflare Pages Très rapide CDN global, DNS intégré, Turnstile, Web Analytics. Statique (ou fonctions Workers si besoin ponctuel). Prod SISA + domaines clients (OSBL/Autonomes/PME).
Netlify Drag & drop ultra simple, previews par PR. Plan gratuit limité; domaine netlify.app. Maquettes rapides si on évite Git au début.
WHC cPanel, PHP/MySQL, email IMAP au Canada. Moins optimisé pour statique; coût récurrent. Sites dynamiques ou besoins email gérés.

Checklist de mise en ligne (statique)

  1. Mettre le site dans un dépôt Git (GitHub).
  2. Activer Pages (GitHub/Cloudflare) — build vide, dossier racine `/`.
  3. Images en WebP/avif si possible, `loading="lazy"`, tailles définies.
  4. Meta SEO + OpenGraph; `sitemap.xml` + `robots.txt`.
  5. Sur Cloudflare : Analytics + (si formulaire) Turnstile; politique CSP simple.
